Test Result

Now we will use POSTMAN to test the results and perform CRUD operations for the user_info table.

  • If you haven’t already, you can download POSTMAN here.

  • Copy the Invoke URL.

  1. Check the status of the API (GET/status):

    • Select the GET method.
    • Paste the copied URL and add /status.
    • Click Send.
    • The status is ready, and this is the response.

  2. Add a new user (POST/user):

    • Select the POST method.
    • Paste the copied URL and add /user.
    • Go to Body -> raw -> JSON.
    • Fill in the user information: 
      {  
    "userid": "1",
    "name": "Tien Dat",
    "email": "TD@gmail.com",
    "age": 21 
}
    • Click Send.
    • Here is the result.
    • Continue adding a few more users!

  3. View the list of users in the DB (GET/users):

    • Select the GET method.
    • Paste the copied URL and add /users.
    • Click Send.
    • Here is the list of users that were added earlier.

  4. Get user information by ID (GET/user?userid={userid}):

    • Select the GET method.
    • Paste the copied URL and add /user?userid={userid}.
    • Click Send.
    • Here is the user information retrieved by ID.

  5. Update user information (PATCH/user):

    • Select the PATCH method.
    • Paste the copied URL and add /user.
    • Go to Body -> raw -> JSON.
    • Fill in the information for the user to be updated: userid, the column to update (age, name, email), and the new value. 
      {  
    "userId": "1",
    "updateKey": "age",
    "updateValue": 10       
}
    • This is the message when the update is successful.

  6. Delete a user (DELETE/user):

    • Select the DELETE method.
    • Paste the copied URL and add /user.
    • Go to Body -> raw -> JSON.
    • Specify the userid you want to delete: 
      {
    "userID": "2"
}
    • This is the message when the deletion is successful.

  7. List of users after updates and deletions.

You have now completed the CRUD lab with API Gateway, Lambda, and DynamoDB. Remember to clean up resources to avoid incurring costs.
